# TicketLock API

**TicketLock API** — это асинхронный бэкенд для системы бронирования билетов на мероприятия, построенный на принципах **Domain-Driven Design (DDD)**.

### 🚀 Основной функционал
*   **Безопасность:** Полноценная асимметричная авторизация (JWT RS256) с использованием приватного и публичного ключей.
*   **Управление пользователями:** Регистрация, аутентификация и ролевая модель доступа (Admin/User).
*   **Мероприятия:** Создание и управление событиями, автоматическая массовая генерация билетов (**Bulk Insert**) для залов любой вместимости.
*   **Бронирование:** Механизм временного резервирования мест с использованием пессимистических блокировок базы данных (**SELECT FOR UPDATE**).
*   **Валидация:** Строгая проверка бизнес-правил внутри доменного слоя (инварианты сущностей).

### 🛠 Стек технологий
*   **Framework:** FastAPI
*   **ORM:** SQLAlchemy 2.0 (Async)
*   **Database:** PostgreSQL
*   **Security:** PyJWT (RSA), Passlib (Bcrypt)
*   **Config Management:** Pydantic Settings (V2)
*   **Infrastructure:** Docker & Docker Compose

### 🏗 Архитектура проекта
Проект разделен на независимые слои согласно DDD архитектуре:
1.  **Domain Layer:** Чистые сущности (Entities) и логика бизнеса. Не зависит ни от каких внешних библиотек.
2.  **Application Layer:** Сервисы и интерфейсы (Use Cases). Оркестрация данных и управление транзакциями через паттерн **Unit of Work**.
3.  **Infrastructure Layer:** Реализация доступа к данным (Repository Pattern), настройки БД и работа с JWT.
4.  **Presentation Layer:** Маршрутизация (Routes), Pydantic схемы (DTO) и внедрение зависимостей.
